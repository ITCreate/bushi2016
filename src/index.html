RFC1149 - 鳥類キャリアによるIPデータグラムの伝送規格

甲村 迅 - ネットワークセキュリティ専攻 1年

概要

鳥類キャリアによるIPデータグラムの伝送規格(英: IP over Avian Carrier, IPoAC)とは、1990年4月1日にIETFがRFC1149として定義したジョーク規格です。その名の通り、伝送媒体に鳩を中心とした鳥類を用いてパケットを伝送する手順を定めています。伝書鳩のようなもの(というか伝書鳩)です。

伝送手順

RFCによれば

  1. 紙巻きに16進数でビットを印字し、オクテットごとに白い部分と黒い部分で区切る
  2. 紙巻きを鳥類キャリアの脚に巻き付ける
  3. 飛ばす

の3つの手順が記されていますが、鳥類を任意の地点に飛行させる事は困難です。 伝書鳩においては下準備として宛先地点を以下の方法で設定する必要があります。

  • 宛先地点を巣と認識させる (巣を作らせる, 鳩舎を作る)

巣を認識させることによって鳥類キャリアは優れた帰巣本能で宛先に直行できます。 ただしIPoACはベストエフォートな為、多少のパケットロス(行方不明)は許容しなければなりません。 また帯域幅は足の長さに左右され、標準的なMTUは256ミリグラムでキャリアの年齢によって増加していきます。

L3PDUを脚に巻き付けている鳩

利点

IPoACには一般に利用されるイーサネットと比べて様々な利点があります。

衝突 : IPoACによる通信では衝突がほぼ発生しません。 イーサネット規格は伝送路が1次元空間であるのに対し、鳥類キャリアは3次元空間を自由に飛び回ることが可能なので、衝突の発生を最小限に抑える事が可能です。

衝突回避 : イーサネットではCSMA/CDによって衝突の検知, 再送制御を行いますが、 鳥類キャリアには衝突回避機能がデフォルト(本能)で備わっています。 広い伝送路と本機能によって衝突の発生を最小限に抑えられます。

伝送距離 : イーサネットでの最大伝送距離は、長いものでは150km(1000BASE-LHB)ほどありますが、鳥類キャリアは1000km以上離れていても伝送が出来ると言われています(キャリアの種類による)。 なお、鳥類キャリアとしての要件を満たすかは不明ですが、キョクアジサシという鳥は32000km飛行し、1年のうちに北極圏と南極圏の間を往き来するようです。

キョクアジサシ

その他 : ワーム(イモ虫)の検知・駆除機能を備えています。 またキャリアは自己修復機能も備えていますが、修復には時間を要します。

問題点

鳥類キャリアには様々な利点がありますが、問題点も複数あります。

超高遅延 : 想像に難くないですが鳥類キャリアはレスポンスが極めて悪いです。 2001年4月、ノルウェーのLinuxユーザグループ(BLUG)がIPoACを実装し、5km離れた宛先に9個のパケット(9羽の鳩)を用いてpingを送信する実験を行いました。 pingが帰ってきたのは1時間〜1時間半後で、帰ってきたのは4羽のみでした。 よって 損失率55%, 応答時間3000秒(3,000,000ms)〜6000秒(6,000,000ms)以上との結果により、実用に耐えないと結論づけています。

Multicast, Broadcast不可 : IPパケットは通常フラッディングによる分身が可能ですが、鳩は物理的に分身が出来ないので、ブロードキャストの定義はされていません。 よって利用できるネットワークタイプはポイントツーポイントに限られます(NBMAも可能？)。

BroadcastStormによるパケットの損失 : ブロードキャストが定義されていない事は前述の通りですが、ブロードキャストストーム(嵐)によってパケットが失われる可能性があります。

おわりに

初部誌にこんな記事を書いて怒られないかと震えていますが、IPoACについて一通りの説明をさせていただきました。間違った事を書いていると思うので、見つけた方はこっそり教えていただけると幸いです。 ちなみに鳥類キャリアが関わるRFCは

  • RFC1149 (鳥類キャリアによるIPデータグラムの伝送規格)
  • RFC2322 (洗濯バサミDHCPによるIPアドレスの管理手法)
  • RFC2549 (サービス品質(QoS)を備えたIPoAC)
  • RFC6214 (IPv6のためのRFC1149の適応)

の4つがあるので、気になる方は読んでみてください。 最後まで読んでいただきありがとうございました。

RFC1149(英): https://tools.ietf.org/html/rfc1149
RFC1149(和): http://www.chibutsu.org/iorin/rfc/rfc1149fr.html
RFC1149(和): http://www.asahi-net.or.jp/~aa4t-nngk/rfc1149.html

ところで、"鳥類キャリアは実用に耐えない"と上で書いていましたが、 伝送速度で伝書鳩に負けるISP(*注)があるらしいので、もしかしたら実用性があるのかもしれませんね。 ないですね、失礼しました。

山崎 好洋 - ITスペシャリスト専攻 3年

シンプルなJavaScriptフレームワークVue.js

一年ほど前、いつまでもjQueryだけではダメだと思いAngularやReactを勉強しようと思いました。 しかしディレクティブやディスパッチャーなどわからない単語が多かったり、公式ページが英語だったりと壁が多く挫折してしまいました。 そんな時に出会ったのがVue.jsです。

Vue.js公式サイト(https://jp.vuejs.org/)

JavaScriptフレームワークの入門に最適！！

いつも周りの人におすすめしているのですが、Vue.jsの基本的な役割は双方向バインディングだけとシンプルなフレームワークなのでJavaScriptフレームワーク初心者の方でも学習しやすいです。 また公式ドキュメントの和訳がしっかりしているので英語が苦手な方でも大丈夫です。

表示したい部分には{{ 変数 }}、フォームにv-mode="変数" とするだけで双方向バインディングができます。 

import Vue from 'vue'

const app = new Vue({
    el: '#app',
    data: {
        msg: 'Hello!'
    }
})

<body>
    <div id="app">
        {{ msg }}
        <input type="text" v-model="msg">
    </div>
</body>

柔軟な拡張性

Vue.jsは双方向バインディングのサポートしかしてないのですが、公式ライブラリなど使用することで様々な規模のアプリケーションを作成することができます。

SPA

vue-routerを利用することで最近流行りのシングルページアプリケーションを作成することができます。

npm install vue-router

Fluxアーキテクチャ

規模が大きくなり複雑なアプリケーションになった場合はFluxライクなライブラリVuexを使います。

npm install vuex

テンプレート、スタイル、スクリプトをまとめてくれるvueファイル

Vue.jsでは.vueという拡張子のファイルを使用することができます。 コンポーネントの要素であるテンプレート、スタイル、スクリプトを一つのファイルにまとめることができます。 

<template>
    ここにテンプレート
</template>

<style scoped>
    ここにスタイル
    scopedをつけるとこのコンポーネント内だけで有効なCSSになります。
</style>

<script>
    ここにスクリプト
</script>

ただvueファイルを使用する場合はwebpackやbrowserifyを利用する必要があります。

コマンド一つでVue.jsを始める

初心者の壁になる一つがGulpなどのタスクランナーやwebpackなどのビルドツールです。 フレームワークを使って何か作りたいのにツールの設定方法などを勉強しないといけないのはもどかしいですよね。 Vue.jsではwebpackの設定やディレクトリ構成などをコマンド一つでしてくれるvue-cliというものがあるのですぐにVue.jsを使うことができます。 

npm install -g vue-cli

vue init webpack-simple#1.0 プロジェクト名

※webpack-simpleのあとに#1.0とつけない場合はvue2.0が入ります。

Vue.jsが新しく！

2016/10/1にVue.jsのバージョンが1.0から2.0になりました！ドキュメントの和訳はまだなのですが（11月中ぐらいには和訳されるらしい） Vue2.0では仮装DOMやサーバーサイドレンダリングなどが実装され速度の向上やSPAならではのSEO問題が解決されました。 またVue1.0との互換性がよくほぼそのままのコードでVue2.0を動かすことができます。

まとめ

シンプルで学習しやすいVue.jsですが万能というわけではありません。Vue.jsからJavaScriptフレームワークを学んでもらい慣れたあとAngulerやReactなどのフレームワークを学習するのがいいのではと私は思っています。

今回KOFに出展しているMeetMapという作品もVue.jsを使用していますので是非見に来てください。 また私のGitHubにてVue.jsで作ったToDoアプリ、Vue2.0のSPAアプリを公開しているので学習の参考に使っていただけるとありがたいです。

出展作品紹介

概要

今回出展させていただいたのは、 「 BookMark Binder 」という、Webアプリです。

これは、 「 お気に入りのリンクをファイルに出力するWebアプリ 」で、画面のフォームからお気に入りのリンクと付けたい名前を記述して登録するだけでファイルにお気に入りのリンクを出力することができます。

アプリによって出力されたファイルを画面から読み込むことでお気に入りを画面に表示させてお気に入りのリンクを利用できます。

機能一覧

  • お気に入りを登録する
  • お気に入りの一覧をファイルに出力する
  • ファイルからお気に入りの一覧を読み出すことが出来る

使用した技術

  • React.js
  • jQuery(アニメーション関係のみ)
  • JavaScript

なぜ作ったの？

常々思っていた、下記の問題を解決したいと思って作りました

  1. 複数のブラウザを使う身からするとお気に入りが共有できない
  2. 勉強した時に使ったWebサイトを まとめて共有・保存 しておきたいときにもう一度リンクを探すのが大変
  3. ナイショのお気に入り とかを作りたいときに家族共有PCとかだと作れない

どうやって解決したか

  • 1 => ファイル化するので、各ブラウザでお気に入りファイルを読みこめば良い
  • 2 => 開いていたタブのリンクをファイルに出力してそのファイルを Slack や GitHub などで共有するようにすればOK
  • 3 => 出力したファイルを誰にも見られないところに保存しておけばOK…

これから追加したい機能

  • お気に入りを編集する機能
  • お気に入りを削除する機能
  • お気に入りに対してコメントを付ける機能

詰まった点・苦労した点

  • React の jsx の記法に悩まされました…
  • この this(props) は何が入ってるんや…

まとめ

出展作品については、初めて React.js を触ったので、React のいいところをきちんと利用できてはいないのですが、ところどころ「あ、これは Reactで書いていてよかった」と思うようなところにも気づけたので大きな収穫だったと思います！
